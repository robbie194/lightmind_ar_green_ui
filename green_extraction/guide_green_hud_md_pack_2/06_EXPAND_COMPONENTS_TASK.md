# 阶段 3.5 任务：在现有 HUD 组件库中扩展“参考图里出现过的组件”（保持同一风格与导出能力）

> 适用场景：你已经有可工作的绿色 HUD 组件页面（可导出 SVG / PNG），现在需要 **在不重构现有架构** 的前提下，继续扩充组件数量，把参考图中出现过的功能入口 / 面板类型补齐到一个更完整的组件库。

---

## 前提（必须）
只有在以下条件满足时才执行本任务：
- 已有项目可以打开使用（`index.html` 正常）
- 已有全局控制区可调（主色 / 线宽 / 发光 / 圆角 / 缩放）
- 已有组件导出 SVG / PNG 功能真实可用
- 已有基础风格已确认（绿色 HUD / AR 叠加风格）

> **禁止重写整个项目**。请基于现有代码增量修改。

---

## 本次任务目标（明确）
在现有页面中新增更多与参考图语义对应的组件，让页面成为一个 **可扩展 HUD 组件库**。  
新增内容需覆盖参考图里出现过的典型场景，包括但不限于：

- 图标卡片类（更多功能入口）
- 行情/股票类面板
- 地图导航叠加层
- 实时翻译/识别结果面板
- 会议记录/会议摘要面板
- 拍摄/相机提示层
- 底部状态栏 / 小型 HUD 状态组件
- 横向提示条（新增变体）

所有新增组件都必须支持导出 **透明背景 SVG / PNG**。

---

## 核心原则（必须遵守）

### 1) 只扩展，不推倒重来
- 保留现有导出函数与导出方式
- 保留现有 token 控制机制（CSS 变量或 JS token）
- 保留现有组件卡片展示方式（预览 + 导出按钮）

### 2) 风格统一高于数量
- 所有新增图标必须遵守已有 stroke-only 规则
- 不允许出现某些图标过粗、某些过细的情况
- 不允许出现拟物、厚填充、重霓虹发光

### 3) 语义借鉴，不复制图形
- 可以学习“这是地图导航”“这是翻译”“这是会议记录”
- 不要直接复刻参考图的具体 SVG path / 完整布局 / 品牌图标
- 品牌名可替换为通用名称（如“管理器”“翻译”“导航”等）

### 4) 导出可靠性不能退化
- 新增组件必须复用并验证导出逻辑
- 不允许出现“预览能看见但导出空白 / 裁切 / 带背景”的情况

## A. 新增图标卡片类（至少新增 12 个）

请在现有图标卡片基础上，新增以下图标卡片（统一风格）：

1. **会议记录**
   - 语义：文档/记录/笔记
   - 图形方向：文档轮廓 + 列表线 / 记事符号
   - 不要做成过于写实的纸张阴影

2. **实时翻译**
   - 语义：翻译 / 双语
   - 图形方向：字母 + 文本框 / 双向切换 / 语言符号
   - 保持几何简洁，不要复杂字符细节

3. **地图导航**
   - 语义：导航 / 路线 / 位置
   - 图形方向：定位点 + 路径折线 / 路牌箭头
   - 适合与导航面板形成语义呼应

4. **AI 助手**
   - 语义：智能助手 / 对话
   - 图形方向：对话框 + AI 标识感（抽象），或简洁机器人头部轮廓
   - 不要做复杂拟人头像

5. **提词器**
   - 语义：提示词 / 演讲辅助
   - 图形方向：屏幕/提词框 + 横线
   - 线条简洁，避免写实支架

6. **歌词**
   - 语义：歌词 / 音乐文本
   - 图形方向：音符 + 文本线 / 歌词卡片
   - 保持 stroke-only

7. **相机拍摄**
   - 语义：拍照 / 快门
   - 图形方向：相机轮廓 + 镜头
   - 与拍摄提示层组件一致风格

8. **视频录制**
   - 语义：录像 / Record
   - 图形方向：摄像机轮廓 / REC 圆点（圆点可仅描边）
   - 不要做填充红点（保持统一主色）

9. **相册/图片**
   - 语义：相册 / 图像查看
   - 图形方向：图片框 + 山峰线 / 相册叠层
   - 简化细节

10. **文件管理**
    - 语义：文件夹 / 目录
    - 图形方向：文件夹轮廓
    - 与“管理器”图区分开（一个偏应用网格，一个偏文件夹）

11. **股票/行情**
    - 语义：行情 / 股票观察
    - 图形方向：折线图 / 蜡烛图（简化）/ 上升趋势
    - 与已有“趋势”图标可不同构图

12. **翻译识图 / OCR**
    - 语义：识别 / OCR / 文字提取
    - 图形方向：取景框 + 文本线 / OCR字样抽象替代
    - 不直接使用品牌含义

> 如果已有同义图标（如“趋势”“主页”“设置”），不要重复；请新增语义不同的图标卡片。

## 技术实现要求（必须保留并扩展）

### 1) 新增组件必须接入全局 token 控制
新增 SVG 组件需要跟随全局控制项变化：
- 主色
- 线宽
- 发光强度
- 圆角
- 缩放（若已实现）

### 2) 新增组件必须复用导出逻辑
- 使用现有 `exportSvg(...)` / `exportPng(...)`（或同类函数）
- 不要复制一套新的导出代码造成分叉
- 若需适配大面板/长条组件尺寸，请在导出函数中增强尺寸读取逻辑（向后兼容）

### 3) glow/filter 处理
如果组件使用 SVG filter 发光：
- `defs/filter` 要包含在组件 SVG 内
- filter 区域要足够大，避免裁切
- 导出 PNG 时发光不能被裁掉

### 4) 文本排版
- 中文文本优先可读性，避免过密
- 面板类组件建议定义字号层级（标题 / 主值 / 次级 / 注释）
- 尽量使用系统字体，不依赖外部字体下载
