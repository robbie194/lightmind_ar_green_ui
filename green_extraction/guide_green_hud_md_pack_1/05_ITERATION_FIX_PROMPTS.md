1. - # 二次迭代修正指令模板（给 Codex）

     ## 用法
     当第一版结果不够像、导出有问题、图标风格不统一时，直接复制下面对应模板给 Codex，并附上当前代码与截图。

     ---

     ## 模板 1：风格不够像参考图（太“普通科技风”）
     请基于现有代码继续修改，不要重写项目。问题是：当前 UI 过于普通科技风，未体现我参考图中的绿色 HUD / AR 叠加风格。
   
     请按以下方向调整（保持导出功能不退化）：
     1. 图标统一改为更纯粹的 stroke-only 风格（减少填充）
     2. 线宽统一并略微变细（参考 STYLE_SPEC）
     3. glow 降低强度，保证线条清晰
     4. 强化 HUD 语法：短分隔线、细线条、圆角框层次
     5. 调整图标卡片中图标与标签的留白比例
     6. 保持现有导出 SVG/PNG 功能可用
   
     请仅输出修改后的文件内容。
   
     ---
   
     ## 模板 2：图标风格不统一（像不同来源拼起来）
     请基于现有代码修正“图标风格统一性”，不要改动页面整体结构。
   
     目标：
     - 所有图标共享统一的绘制规则（视口、线宽、linecap、linejoin、留白）
     - 图标视觉重心统一
     - 图标在卡片中的占比统一
     - 保留各图标语义（设置、图表、管理器等）

     请先简要说明你将统一哪些规则，再输出修改后的代码文件。
   
     ---
   
     ## 模板 3：SVG/PNG 导出不稳定或背景不透明
     请只修复导出功能相关问题，不要改 UI 视觉样式。
   
     当前问题（按实际填写）：
     - [ ] SVG 导出包含页面背景
     - [ ] PNG 导出有黑底/白底
     - [ ] PNG 导出尺寸不对
     - [ ] glow 被裁切
     - [ ] 导出按钮导出的不是目标组件本体
   
     修复要求：
     1. 每个组件导出内容仅包含其内部 SVG
     2. SVG 导出透明背景
     3. PNG 导出透明背景，默认 2x
     4. 正确处理 viewBox / width / height
     5. 若使用 filter，确保 defs/filter 随组件导出且不被裁切
   
     请重点展示并注释：
     - `exportSvg(...)`
     - `exportPng(...)`
   
     ---
   
     ## 模板 4：先扩展组件数量（在风格已稳定后）
     请在现有风格规范和代码基础上扩展组件，保持完全一致的视觉风格与导出能力，不要重构已有导出逻辑。
   
     新增内容：
     1. 新增图标卡片 4 个（与现有风格一致）
     2. 新增一个小状态面板（例如网络/电量/时间综合）
     3. 新增一个细长底部状态栏组件
     4. 所有新增组件都要有 SVG/PNG 导出按钮
   
     请输出修改后的文件内容，并标注新增部分。
   
     ---

     2. 如何按优先级修正（P0/P1/P2）
     3. 每项修正会改哪些 CSS 变量或 SVG 结构
   
     等我确认后，你再输出代码修改。
