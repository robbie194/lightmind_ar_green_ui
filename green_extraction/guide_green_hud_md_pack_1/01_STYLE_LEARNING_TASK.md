# 阶段 1 任务：根据参考图提炼绿色 HUD 风格规范（先不要写代码）

## 任务目标
你将收到多张参考图（来自第三方界面截图）。请基于这些参考图，提炼一套 **可执行、可复用、可扩展** 的绿色 HUD / AR UI 风格规范，用于后续生成原创图标和 UI 组件。

> 重点：是“学习风格语法”，不是“复制具体图形”。

---

## 本阶段禁止事项（必须遵守）
- 不要输出 HTML/CSS/JS 代码
- 不要直接复刻参考图布局
- 不要复制参考图里的图标轮廓路径
- 不要只写抽象描述（如“科技感”“未来感”），必须落到可执行规则

---

## 输入材料
你会收到：
1. 多张参考图（图片文件）
2. `02_REFERENCES_INDEX_TEMPLATE.md`（由我填写每张图的用途说明）

---

## 你需要完成的输出（必须输出 2 份文档）

### 输出 1：`STYLE_SPEC.md`
内容必须包含以下章节（不可省略）：

#### 1. 风格总结（Summary）
- 用简洁语言总结这些参考图的共同视觉特征
- 区分“核心风格特征”和“可变特征”

#### 2. 设计 Tokens（Design Tokens）
请给出可执行参数（建议用表格或代码块）：
- 颜色（主色、弱化线、辅助发光、半透明填充）
- 线宽（主描边、细分隔线）
- 圆角（小组件、卡片、长条面板）
- 发光强度（弱/中/强）
- 间距（内边距、图标与文字距离、分隔线距离）
- 字体建议（系统字体即可）
- 字号层级（标题/主数值/辅助文本）

#### 3. 图标绘制规则（非常重要）
必须明确：
- 是否 stroke-only（建议是）
- `fill` / `stroke` 的使用约束
- `stroke-linecap` / `stroke-linejoin`
- 默认图标视口（如 24x24 / 32x32）
- 图标留白比例（例如 12%~16%）
- 图标复杂度（简洁几何，不拟物）
- 统一视觉粗细规则
- 图标在卡片中的占比（例如 40%~55%）

#### 4. HUD 边框与面板规则
必须明确：
- 圆角矩形边框语法
- 分隔线使用方式（短横线、细线、虚线）
- 是否允许半透明填充（允许但轻）
- glow 的使用位置和强度建议
- 边框与内容的留白关系
- 面板层级结构（标题、主信息、辅助信息、图表）

#### 5. 组件风格映射（根据参考图）
请根据参考图用途，归纳：
- 哪些参考图主要贡献“图标卡片风格”
- 哪些贡献“信息面板布局”
- 哪些贡献“横向提示条布局”
- 哪些贡献“状态栏小图标风格”
- 哪些贡献“发光/线条质感”

> 仅做风格映射与总结，不要照搬。

#### 6. Do / Don’t（边界定义）
明确列出：
- 必须像什么（例如：细线 HUD、轻发光、圆角框）
- 不要做成什么（例如：重霓虹、厚实心、彩色赛博朋克）

#### 7. 风险与不确定性（可选但建议）
如果参考图风格存在冲突，请指出并提出取舍建议。

---

### 输出 2：`COMPONENT_PLAN.md`
用于阶段 2 编码前的实现规划。必须包含：

#### 1. 第一版最小可用组件（MVP）
至少包含：
- 图标卡片类（4个）
- 信息面板类（1个）
- 横向提示条类（1个）
- 状态栏元素（1组）

#### 2. 每个组件的结构定义
例如：
- 图标卡片 = 外框 + 图标 + 分隔线 + 标签 + 导出按钮（按钮是页面UI，不是导出内容）
- 信息面板 = 标题 + 名称代码 + 主数值 + 涨跌文本 + 折线图 + 时间刻度

#### 3. 样式可调项（全局控制区）
至少包含：
- 主色
- 线宽
- glow 强度
- 圆角
- （可选）缩放倍率

#### 4. 导出方案概要
简述：
- SVG 导出思路
- PNG 导出思路（SVG -> Canvas）
- 透明背景保证方式

- 先输出 `STYLE_SPEC.md`
- 再输出 `COMPONENT_PLAN.md`
- 两份文档都使用 Markdown 格式
- 内容必须具体，不要只写概念性词汇

---

## 评价标准（你需要自检）
你输出的规范必须满足：
- 工程师可以按规范直接写 UI（不需要再猜）
- 图标风格与面板风格规则明确
- 可支持后续扩展更多 HUD 组件
- 不依赖复制原图路径
