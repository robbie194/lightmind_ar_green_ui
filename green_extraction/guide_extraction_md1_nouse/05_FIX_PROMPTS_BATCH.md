# 批量提取方案迭代修正指令模板（给 Codex）

## 用法
当第一版“自动提取绿色部分”的结果有漏检/误检/边缘不好/矢量化粗糙时，直接复制对应模板给 Codex，并附上当前代码与部分输出截图/日志。

## 模板 1：漏检太多（绿色没被提出来）
请基于现有批量提取脚本继续修改，不要重写整个项目。当前问题是：漏检较多，很多绿色UI元素没有被识别出来。

请重点从以下方向优化（保持透明PNG导出逻辑不退化）：
1. 扩展绿色HSV阈值范围，并支持“双阈值合并”（核心亮绿 + 弱绿发光）
2. 降低过严的 `s_min` / `v_min` 默认值（但仍避免背景噪声）
3. 调整形态学参数以连接断裂线条（如适当闭运算）
4. 增加 `padding` 默认值，避免导出时截断发光边缘
5. 在 `summary/debug` 中输出漏检分析辅助信息（如每图绿色像素占比）

请仅输出修改后的文件内容，并说明默认参数如何变化。

## 模板 2：误检太多（背景反光/噪点也被提取）
请在现有批量提取脚本基础上优化“误检过滤”，不要改动整体目录结构与输出格式。

当前问题：
- 提取了大量无意义绿色噪点/反光
- 候选过多，影响后续筛选效率

请优化：
1. 加强小连通域过滤（面积阈值）
2. 增加最小宽高阈值
3. 增加绿色占比过滤（候选bbox内mask占比）
4. 增加长宽比过滤（过细/异常形状可过滤）
5. （可选）增加边缘密度或连通性评分作为质量分

保持：
- 透明PNG导出逻辑不变
- `candidates.json` / `summary.json` 格式兼容
