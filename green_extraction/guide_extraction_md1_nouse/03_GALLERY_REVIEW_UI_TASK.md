# 阶段 2：生成本地 Gallery 预览页（快速查看自动提取结果，不逐张看原图）

## 任务目标
生成一个 **纯静态 HTML/CSS/JS 本地预览页**，用于快速查看阶段 1（和阶段 3）输出的候选素材，而不是逐张查看原始参考图。

> 用户目标是减少人工工作量：自动提取后，只对候选结果做快速浏览和少量筛选。

---

## 输入数据
默认读取以下内容（按实际存在自动适配）：
- `green_extract_output/metadata/candidates.json`
- `green_extract_output/metadata/summary.json`
- `green_extract_output/metadata/vectors.json`（如存在）
- `green_extract_output/crops_png/`
- `green_extract_output/vectors_svg/`（如存在）

---

## 必须实现的页面功能

### 1) 概览统计区（顶部）
显示至少：
- 原图总数
- 候选总数
- 过滤数量（如有）
- 成功矢量化数量（如有）
- 当前筛选结果数量

---

### 2) 筛选控制区（必须）
至少实现以下筛选中的 3 项：
- 按来源文件名搜索（文本搜索）
- 按尺寸范围筛选（宽/高）
- 按面积范围筛选（像素面积）
- 仅显示已矢量化 / 未矢量化
- 仅显示大于某质量分（如有score）

> 目标是减少“逐张看原图”的负担，因此筛选区很重要。

---

### 3) 候选网格区（核心）
每个候选卡片至少显示：
- 透明背景预览（建议用棋盘格背景便于看透明）
- 候选文件名
- 来源原图文件名
- bbox/尺寸信息（简略）
- 面积信息（简略）
- 是否有SVG版本（标识）

#### 卡片操作（至少实现 2 项）
- 打开对应 PNG（新标签页或下载）
- 打开对应 SVG（若存在）
- 复制文件名（可选）
- 隐藏此项（前端临时）
- 标记保留/删除（前端临时，不要求落盘）

---

### 4) 视图模式（建议）
- 网格视图（默认）
- 列表视图（可选）
- 大图预览模式（可选）

---

## 技术要求（必须）
- 纯 HTML + CSS + 原生 JS
- 不依赖后端
- 尽量从本地 JSON 文件加载
- 若浏览器对 `file://` 读取 JSON 有限制，请在 README 中提供本地静态服务器方式（如 `python -m http.server`）

---

## 目录结构建议
```text
review_gallery/
  index.html
  styles.css
  script.js
  README.md
```

---

## 数据加载要求（必须）
- 优先读取 `candidates.json`
- 若 `vectors.json` 存在，建立候选到SVG的映射
- 对缺失文件做容错提示（不要页面直接崩溃）

---

## 透明预览显示（重要）
为方便查看透明PNG，卡片预览区域建议：
- 使用棋盘格背景（CSS）
- 居中显示图像
- 限制最大显示尺寸，防止超大图撑坏布局

---

## 可选增强（建议）
1. **批量导出清单**
   - 把用户“保留”的候选导出为 `keep_list.json`
2. **质量排序**
   - 按面积、尺寸、score 排序
3. **相似项分组**
   - 按来源图分组展示
4. **快速键盘操作**
   - `K` 保留 / `H` 隐藏 / `→` 下一项（可选）
