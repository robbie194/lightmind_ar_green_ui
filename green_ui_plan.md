# LightMind AR 绿色 UI 重建与导出方案（不抠图）

## 目标

- 参考 `leqi_ui/` 截图里“绿色覆盖层”的图案/样式（线框、圆角框、图标、HUD 元素等），不需要背景画面。
- 不做抠图、不追求与对方绿色像素级一致；用“识别结构 + 自己重建”的方式做一套 LightMind 的绿色 UI kit。
- 做一个**静态 HTML 页面**（无构建也能跑）：
  - 能看到所有“绿色图案/组件”（SVG 组件库 + 模板）。
  - 能把单个组件或组合后的面板**导出为透明背景 SVG**，并支持**导出透明 PNG**。

## 现状与约束（基于当前目录素材）

- `leqi_ui/` 下是多张 `jpg` 截图：它们只作为“风格参考/结构参考”，不作为素材来源。
- 我们的交付物应该是**自有 SVG 组件**与**组合模板**，避免像素级复制。
- 纯静态页面要实现“导出 PNG”，需要在浏览器里把 SVG 渲染到 canvas（用纯前端库即可）。

## 实现路线（推荐按顺序做）

### 路线 1：先做 LightMind 绿色 UI kit（核心产物）

**定位**：完全不依赖截图抠图，直接产出一套可复用的 HUD 组件库 + 组合模板。

**核心思路（组件化 + 设计参数化）**

1. 定义一组 LightMind “设计 tokens”（CSS 变量或 JSON）：
   - `--lm-green`（主绿）、`--lm-green-dim`（弱化绿）
   - `--lm-stroke`（线宽）、`--lm-radius`（圆角）、`--lm-gap`（间距）
   - `--lm-glow`（发光强度，SVG filter）
2. 用纯 SVG 画出基础图元（可组合）：
   - 圆角矩形框（支持内外双线、角标、分割线）
   - 圆环/准星/定位点
   - 进度条、信号条、电量条（可参数驱动）
   - 常用 icon（齿轮、WiFi、电量、返回、网格等）: 用自绘路径，避免依赖字体图标
3. 基于图元拼装“模板”（一屏 UI 结构），参考 leqi 的布局语法但内容/文案使用 LightMind：
   - `StatusBar`：时间、温度、电量、WiFi
   - `AppGrid`：一排/多排的圆角卡片 + icon + label
   - `InfoCard`：大信息框（多行文字、提示）
   - `Toast`：底部提示条
4. 所有组件都输出为 SVG（透明背景），并且可在页面里：
   - 单独预览/下载
   - 拖拽到“画布”组合成一个面板
   - 统一改色（改 token 即可）

**页面功能（静态 HTML）**

- `Gallery`：所有组件与模板的缩略图列表（SVG 渲染）
- `Composer`：一个画布区域，支持把组件放进去（基础：点击添加；进阶：拖拽、缩放、对齐）
- `Tokens`：右侧参数面板，实时调整 stroke/radius/glow/颜色
- `Export`：
  - 导出当前选中组件为 `*.svg`
  - 导出当前画布组合为 `panel.svg`
  - 导出 PNG（透明）：把 SVG 渲染到 canvas 后 `toBlob('image/png')`

**优点**

- 彻底避免抠图与 JPEG 噪声问题。
- 输出物是“可编辑/可缩放”的矢量组件，更适合你后续在 AR 里实时渲染。
- 风格相似但不复制：只借鉴布局语法与视觉语言（圆角、线宽、发光、间距）。

**缺点**

- 需要先做一套组件库（比抠图更“设计工程化”）。

### 路线 2：用 OCR/识别辅助“理解参考 UI”（可选，但省手工）

**定位**：不是提取像素，而是用 OCR/检测来“读懂”参考截图里有哪些信息块，帮我们快速生成模板草稿。

**推荐方式：纯前端 OCR（不需要系统安装）**

1. 在 `ui/reference.html` 里加载 `leqi_ui/*.jpg`（只本地使用，不入库）。
2. 用 `tesseract.js` 在浏览器做 OCR：
   - 抓到时间/电量/温度/菜单文案等文字
   - 得到文字 bounding boxes，推断大概布局区域
3. 结合极简的几何检测（可选）：
   - 找长直线/大圆角矩形的外框（用简单的边缘检测/阈值也行，要求不高）
4. 生成一个 `template.json`（只存结构，不存截图）：
   - blocks: statusBar/appGrid/infoCard/toast
   - 每个 block 有位置、尺寸、内容字段
5. `ui/index.html` 读取这些 `template.json`，用自有 SVG 组件渲染成 LightMind 风格的模板。

**输出物**

- `templates/*.json`：结构化模板（可提交到 repo）
- `ui/`：静态页面（渲染模板、组件库、导出）
- `leqi_ui/`：只本地参考，不提交

## 关于“参考而不拷贝”的实现策略

你要的是“参考 Leqi 的绿图案风格”，更稳妥的做法是：

- 把参考 UI 当成“版式与视觉语言教材”，提取的是：线宽、圆角、留白、发光、信息层级。
- 用自有 SVG 组件库重建：同样的语法，但换成 LightMind 图标/文案/结构。

## 我准备怎么在这个仓库里落地（实现步骤）

1. 做 `ui/index.html`（静态）：
   - 组件库 Gallery（SVG）

   - 导出 SVG/PNG（透明）
2. 做 `ui/components/*.js`（或单文件也行）：

3. 做 `templates/presets/*.json`：
   - `statusbar.json` / `appgrid.json` / `infocard.json`
4. 可选：`ui/reference.html` + `tesseract.js`：
   - 读取本地参考图，半自动生成模板 JSON

## 验收标准（你一眼能判断对不对）

- 打开 `ui/index.html`，能看到一堆“LightMind 绿色 HUD 组件”，背景透明或棋盘格。
- 任意选择一个组件或模板，导出得到透明 `svg`。
- 导出的 PNG 放到任意背景上不带底色，发光与线条仍清晰。

## 需要你确认的 3 个点（我开始写代码前）

1. 你要的交付物优先级：
   - A. 一套可复用组件（icon/框/状态栏等）
   - B. 一套整屏模板（直接像“桌面/菜单/提示框”）
2. 导出优先级：
   - A. SVG（主目标，透明底，方便后续编辑/嵌入 AR）
   - B. PNG（需要的话再做，透明底）
3. LightMind 的主绿（HEX）有没有定值？如果没有，我先给一个默认绿，并在 Tokens 面板里可调。
