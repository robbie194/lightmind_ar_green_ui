# COMPONENT_PLAN.md

## 1. 第一版最小可用组件（MVP）

- 图标卡片类（4 个）：
- `Card-Settings`（设置语义）
- `Card-Navigation`（导航语义）
- `Card-Translate`（翻译语义）
- `Card-Media`（媒体/音频语义）
- 信息面板类（1 个）：
- `Panel-InfoTrend`（标题 + 主数值 + 涨跌 + 简化折线）
- 横向提示条类（1 个）：
- `Banner-Reminder`（左图标 + 双行文本 + 中间圆点装饰 + 底部状态）
- 状态栏元素（1 组）：
- `StatusBar-Core`（时间、温度、WiFi、电池）

## 2. 组件结构定义

### `Card-*` 图标卡片结构

- 外框：圆角矩形，单线描边，低强度 glow。
- 主图标：stroke-only 几何图标，位于上半区中心。
- 分隔线：短横细线，位于图标与标签之间。
- 标签：单行短文本，居中。
- 导出注意：控制区按钮不属于组件 DOM，不参与导出。

### `Panel-InfoTrend` 信息面板结构

- 顶栏：左标题 + 右时间。
- 主区：名称/代码 + 主数值 + 次级变化值。
- 图表区：单条折线 + 一条弱化基准线 + 两端时间刻度。
- 背景：可选轻填充，优先保证文字和折线清晰。
- 对齐：文本左对齐，图表水平铺开。

### `Banner-Reminder` 横向提示条结构

- 外框：长条圆角矩形，单线或双线（默认单线）。
- 左侧：小图标 + 两行文本（标题/副标题）。
- 中部：圆点装饰或交互占位（纯视觉，不导出操作按钮）。
- 底部：简化状态信息（时间/温度或进度文案）。

### `StatusBar-Core` 状态栏结构

- 左区：时间 + 温度（同基线）。
- 右区：WiFi 图标 + 电池图标 + 电量文本。
- 图标与文本按基线对齐，间距遵循 token。

## 3. 样式可调项（全局控制区）

- 主色：`hud-green-main`。
- 线宽：`stroke-main` / `stroke-sub`。
- glow 强度：`weak` / `mid` / `strong`。
- 圆角：`radius-sm` / `radius-md` / `radius-lg`。
- 缩放倍率：组件级 `0.8x ~ 1.4x`。
- 透明度：`panel-fill-alpha`、`dim-text-alpha`。

## 4. 导出方案概要

- SVG 导出：
- 每个组件保持独立 SVG 根节点。
- 导出时序列化目标组件 SVG（含 `defs/filter`）。
- 组件边界由 `viewBox` 精确限定，避免导出控制区元素。
- PNG 导出：
- 将组件 SVG 以 Blob URL 载入 Canvas，再 `toBlob('image/png')`。
- Canvas 清空为透明背景，不填充底色。
- 透明背景保证：
- SVG 根节点不设置不透明背景矩形。
- 仅组件内部使用透明 fill 和描边。
- glow/filter 防裁切：
- `viewBox` 外扩安全边距（建议 8~12px）。
- `filterUnits="userSpaceOnUse"` 并扩展 filter 区域。
- 不导出控制按钮机制：
- 控制区 DOM 与可导出组件 DOM 分离。
- 导出时只选取组件容器节点，不序列化页面 UI 控件。

## 5. 开发顺序建议

1. 建立全局 tokens 与基础 SVG `defs` 规则。
2. 实现单个图标卡片与单组件 SVG/PNG 导出链路。
3. 扩展到 4 个图标卡片并复用同一绘制语法。
4. 实现信息面板 `Panel-InfoTrend`（含折线与层级文本）。
5. 实现 `Banner-Reminder` 与 `StatusBar-Core`。
6. 接入全局样式控制区并绑定实时更新。
7. 做导出边界与透明度自检，修复 filter 裁切问题。

## 6. 第一轮验收口径（阶段 1 文档层）

- 工程师可直接据此进入编码，不需补充风格猜测。
- 组件类型覆盖图标卡片、信息面板、提示条、状态栏四类核心形态。
- 导出策略明确说明 SVG/PNG 透明背景与防裁切方法。
- 所有组件定义基于风格语法，不依赖参考图具体内容复制。
