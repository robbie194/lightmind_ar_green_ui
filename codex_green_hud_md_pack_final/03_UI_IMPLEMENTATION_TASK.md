## 导出功能（必须真实可用）

### A. 导出 SVG（每个组件都必须有）
要求：
- 导出目标组件本身对应的 SVG
- 背景透明
- 不包含页面背景和按钮 UI
- 文件名有意义（例如 `hud-card-settings.svg`）

推荐实现：
- 每个组件的导出内容用单独 `<svg>` 包裹
- 点击导出时序列化该 SVG：`new XMLSerializer().serializeToString(svg)`
- Blob 下载为 `.svg`

---

### B. 导出 PNG（每个组件都必须有）
要求：
- 背景透明
- 默认至少 2x 清晰度
- 不包含页面背景和按钮 UI

推荐实现（优先）：
1. 序列化 SVG 字符串
2. 创建 Blob URL
3. 加载到 `Image`
4. 绘制到 `Canvas`（按 scale 放大，例如 `2x`）
5. `canvas.toBlob(..., "image/png")` 下载

必须处理的技术点：
- Canvas 尺寸按 SVG 的 `viewBox` 或 `width/height` 设置
- 保持透明背景（不要填充底色）
- 若使用 SVG filter 发光，确保滤镜定义在该 SVG 内部
- 避免 glow 被裁切（扩大 filter 区域）
- 正确释放 Blob URL（避免内存泄漏）
