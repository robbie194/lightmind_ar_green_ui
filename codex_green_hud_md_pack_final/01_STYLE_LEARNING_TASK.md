# 阶段 1 任务：根据参考图提炼绿色 HUD / AR 风格规范（先不要写代码）

## 任务目标
你将收到多张参考图（第三方界面截图 / 实拍叠加截图）。请基于这些参考图，提炼一套 **可执行、可复用、可扩展** 的绿色 HUD / AR UI 风格规范，用于后续生成原创图标与 UI 组件。

> 重点：学习“风格语法”，不是复制“具体图形”。

---

## 本阶段禁止事项（必须遵守）
- 不要输出 HTML / CSS / JS 代码
- 不要直接复刻参考图布局
- 不要复制参考图里的图标轮廓路径
- 不要只写抽象词（如“科技感”“未来感”），必须落到可执行规则
- 不要把参考图里的实际股票名称、地点、人物、品牌内容当成最终 UI 内容直接复用

---

## 输入材料
你会收到：
1. 多张参考图（图片文件）
2. `02_REFERENCES_INDEX_FILLED.md`（已填写每张图的用途与优先级）

---

## 你需要完成的输出（必须输出 2 份文档）

### 输出 1：`STYLE_SPEC.md`
内容必须包含以下章节（不可省略）：

#### 1. 风格总结（Summary）
- 用简洁语言总结这些参考图的共同视觉特征
- 区分“核心风格特征”和“可变特征”
- 说明“哪些元素属于 HUD 语法，哪些只是参考图业务内容”

#### 2. 设计 Tokens（Design Tokens）
请给出**可执行参数**（建议表格或代码块），至少包含：
- 颜色（主色、弱化线、辅助发光、半透明填充）
- 线宽（主描边、细分隔线、小图标线宽）
- 圆角（小组件、卡片、长条面板）
- 发光强度（弱 / 中 / 强）
- 间距（内边距、图标与文字距离、分隔线距离）
- 字体建议（系统字体即可）
- 字号层级（标题 / 主数值 / 辅助文本 / 标签）
- 透明度建议（panel fill / dim text / outline）

#### 3. 图标绘制规则（非常重要）
必须明确：
- 是否 stroke-only（建议是）
- `fill` / `stroke` 的使用约束
- `stroke-linecap` / `stroke-linejoin` 推荐值
- 默认图标视口（例如 24x24 / 32x32）
- 图标留白比例（例如 12%~18%）
- 图标复杂度（几何简洁，不拟物）
- 统一视觉粗细规则（避免不同图标粗细不一致）
- 图标在卡片中的占比（例如 40%~55%）

#### 4. HUD 边框与面板规则
必须明确：
- 圆角矩形边框语法（单线 / 双线 / 角标 / 缺口的使用建议）
- 分隔线使用方式（短横线、细线、虚线）
- 半透明填充是否允许（允许但轻）
- glow 的使用位置和强度建议
- 边框与内容留白关系
- 面板层级结构（标题、主信息、辅助信息、图表）
- 文字与图形的对齐方式建议（左齐 / 中齐 / 底对齐）

#### 5. 组件风格映射（根据参考图）
请根据 `02_REFERENCES_INDEX_FILLED.md`，归纳：
- 哪些参考图主要贡献“图标卡片风格”
- 哪些贡献“信息面板布局”
- 哪些贡献“横向提示条布局”
- 哪些贡献“状态栏小图标风格”
- 哪些贡献“发光 / 线条质感”
- 哪些可作为反例（例如：不适合学习的过重遮罩或复杂背景干扰）

> 仅做风格映射与总结，不要照搬。

#### 6. Do / Don’t（边界定义）
明确列出：
- 必须像什么（例如：细线 HUD、轻发光、圆角框、低饱和透明覆盖）
- 不要做成什么（例如：重霓虹、厚实心面板、彩色赛博朋克、拟物按钮）

#### 7. 风险与不确定性（建议）
如果参考图之间风格存在冲突（例如线宽不一致、遮罩强度差异大），请指出并提出取舍建议（主风格 + 次风格）。

---

### 输出 2：`COMPONENT_PLAN.md`
用于阶段 2 编码前的实现规划。必须包含：

#### 1. 第一版最小可用组件（MVP）
至少包含：
- 图标卡片类（4个）
- 信息面板类（1个）
- 横向提示条类（1个）
- 状态栏元素（1组）

#### 2. 每个组件的结构定义（建议用列表）
示例格式（仅示例）：
- 图标卡片 = 外框 + 图标 + 分隔线 + 标签 + 导出按钮（按钮不在导出 SVG 中）
- 信息面板 = 标题 + 名称代码 + 主数值 + 涨跌文本 + 折线图 + 时间刻度
- 提示条 = 长条框 + 左图标 + 双行文本 + 底部状态信息 + 圆形装饰点
- 状态栏 = 时间 + 温度 + WiFi + 电池（图标线框）

#### 3. 样式可调项（全局控制区）
至少包含：
- 主色
- 线宽
- glow 强度
- 圆角
- （可选）缩放倍率
- （可选）透明度 / 填充强度

#### 4. 导出方案概要（必须说明）
简述：
- SVG 导出思路（序列化单组件 SVG）
- PNG 导出思路（SVG -> Canvas，透明底）
- 透明背景保证方式
- glow/filter 防裁切策略
- 不导出页面控制按钮的机制

#### 5. 开发顺序建议（必须有）
建议列出实现顺序，例如：
1. Tokens 与基础 SVG defs
2. 单个图标卡片 + 导出逻辑
3. 批量图标卡片复用
4. 信息面板与折线图
5. 提示条 / 状态栏
6. 控制区绑定全局变量
7. 自检与修正
